# Red Sea Marine Data Catalog

A static catalog of marine datasets covering the Red Sea basin — browse/search, a
spatial footprint map, a coverage gap matrix, and per-dataset detail pages.
Built with Next.js (App Router, static export), Tailwind, MapLibre GL, and
browser-side semantic search via transformers.js. Deployed to GitHub Pages.

Live site: https://integrated-reef-fisheries-lab.github.io/red-sea-catalog/

## Requirements

- Node.js 20+ (CI runs Node 22)
- npm

## Setup

```bash
npm install
```

## Local development

```bash
npm run dev
```

Opens the site at `http://localhost:3000`. The search index (`public/search-index.json`)
is not regenerated by `next dev` — run `npm run prebuild` (or `npm run build`) at
least once first if it doesn't already exist, or after editing any file in
`data/sources/`.

## Building for production

```bash
npm run build
```

This runs two steps:

1. `prebuild` — `scripts/build-search-index.mjs` reads every record in
   `data/sources/`, embeds it with `@huggingface/transformers`
   (`Xenova/all-MiniLM-L6-v2`), and writes `public/search-index.json`.
   **Never hand-edit that file** — it's fully regenerated on every build.
2. `next build --webpack` — statically exports the site to `out/`
   (`output: 'export'` in `next.config.js`). No server runtime, no API
   routes — every page is prerendered HTML.

The `--webpack` flag is required: Next 16 defaults to Turbopack, but the
`onnxruntime-node` alias in `next.config.js` (needed so the browser bundle
doesn't try to pull in transformers.js's Node-only native backend) is a
webpack-specific option.

## Deployment

Pushing to `main` triggers `.github/workflows/deploy.yml`, which runs
`npm ci && npm run build` and publishes `out/` to the `gh-pages` branch via
`peaceiris/actions-gh-pages`. GitHub Pages is configured (Settings → Pages)
to serve from that branch. No manual deploy steps needed beyond pushing.

## Admin tool (editing the dataset)

The "database" is just JSON files in `data/sources/`, one per dataset,
following the schema in [`src/lib/types.ts`](src/lib/types.ts). You can hand-edit
those files directly, or use the local admin UI:

```bash
npm run admin
```

Opens at `http://localhost:4000`. It lists all sources and lets you create,
edit, and delete records through a form instead of writing raw JSON by hand.

**Important:** this is a local-only tool. It runs a small Node server
(`scripts/admin-server.mjs`) that reads and writes `data/sources/*.json`
directly on your filesystem — it is never bundled into the static site and
has no equivalent in the GitHub Pages deployment. To publish changes made
through the admin tool, commit the resulting JSON files and push:

```bash
npm run admin        # make your edits in the browser, then Ctrl+C to stop
git add data/sources
git commit -m "Update dataset records"
git push              # triggers the deploy workflow
```

Notes on the admin UI:

- **ID is immutable** once a source is created (it's also the JSON filename).
- Most fields have plain inputs, checkboxes, or newline/comma-separated text
  areas for arrays (themes, keywords, known issues, etc.).
- The `resources` field (nested resource/variable records) is edited as raw
  JSON in a textarea — see the placeholder hint in the form for the expected
  shape.
- Server-side validation checks required fields (`id`, `title`, `abstract`,
  `domain`, `spatial.bbox`, `temporal.start`, `access.tier`,
  `quality.status`) before writing to disk; errors show inline in the form.

## Project structure

```
data/sources/          One JSON file per dataset — the canonical "database"
public/search-index.json   Generated at build time, never hand-edit
src/app/                Pages: / (browse+search), /map, /coverage, /sources/[id]
src/components/         SearchBar, SourceCard, FacetPanel, CoverageMatrix, MapView, ...
src/lib/types.ts        Canonical TypeScript types mirroring the source schema
src/lib/sources.ts      Build-time (fs-based) loading of data/sources/*.json
src/lib/search.ts       Browser-side semantic search (lazy model load + cosine similarity)
scripts/build-search-index.mjs   Prebuild step that generates public/search-index.json
scripts/admin-server.mjs         Local-only admin server (see above)
admin/                  Static HTML/CSS/JS for the admin UI (served by admin-server.mjs)
```

## Adding a new dataset

Either use `npm run admin`, or add a JSON file to `data/sources/` by hand
following the schema in `src/lib/types.ts`, then run `npm run build` (or just
`npm run prebuild`) to fold it into the search index. New source pages are
picked up automatically at build time via `generateStaticParams` in
`src/app/sources/[id]/page.tsx` — no other code changes needed.
